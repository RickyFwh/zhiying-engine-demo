# 智营引擎 Sprint 计划 - v1.1

## 目标
基于 BRD 差距分析，补齐核心功能，提升产品完整度

## Sprint 1 任务清单（当前）

### P0 - 核心功能补齐

#### 1. 数据看板真实化
- **现状**: Dashboard 使用静态 mock 数据
- **目标**: 追踪真实使用情况（生成次数、Token 消耗、平台分布）
- **文件**: `app/dashboard/page.tsx`, 新建 `lib/analytics.ts`
- **工作量**: 中

#### 2. 审核反馈闭环
- **现状**: 审核队列仅 UI 展示，无实际流程
- **目标**: 专家修改内容后，自动优化提示词库
- **文件**: `app/review/page.tsx`, `app/api/review/route.ts`
- **工作量**: 中

#### 3. 违禁词检查系统
- **现状**: 依赖 LLM 自带过滤
- **目标**: 内置违禁词库，生成前/后双重检查
- **文件**: 新建 `lib/compliance.ts`, 更新内容生成 API
- **工作量**: 小

#### 4. 内容持久化存储
- **现状**: 生成内容仅在前端展示，刷新即丢失
- **目标**: 存储到 localStorage/数据库，支持历史查看
- **文件**: 新建 `lib/storage.ts`, 更新生成页面
- **工作量**: 中

### P1 - 体验优化

#### 5. 竞品分析结构化
- **现状**: LLM 输出纯文本
- **目标**: 解析为结构化数据，展示图表和对比
- **文件**: `app/pipeline/page.tsx`, 竞品分析步骤
- **工作量**: 中

#### 6. 多语言支持（可选）
- **现状**: 仅中文
- **目标**: 支持中英文切换
- **文件**: 新建 `lib/i18n.ts`, 更新所有页面
- **工作量**: 大（可延后）

## 验收标准
- [ ] Dashboard 展示真实使用数据
- [ ] 审核队列支持编辑和反馈
- [ ] 内容生成自动检查违禁词
- [ ] 生成内容可持久化存储和查看
- [ ] 竞品分析结果可视化

## 团队分工
- **产品经理**: 细化需求、定义交互、验收功能
- **研发工程师**: 实现功能、代码审查、技术文档
- **测试工程师**: 编写测试用例、验证功能、报告 Bug

# 智营引擎 - 产品交互逻辑梳理

> 基于 BRD V1.0 文档 + 当前 MVP 实现状态  
> 梳理时间：2026-06-21

---

## 一、BRD 定义的五大核心模块

| 模块 | BRD 定义 | 当前状态 |
|------|---------|---------|
| **模块一：专家决策大脑** | 将专家决策逻辑编码为规则引擎，实时接收数据，输出投放策略 | ✅ 已实现（规则引擎 + LLM 增强） |
| **模块二：内容生成器** | 基于品牌知识批量生成多渠道营销内容（文/图/视频） | ⚠️ 部分实现（文案生成，图片/视频待做） |
| **模块三：投放连接器 (RPA)** | RPA 模拟人工操作，自动管理广告平台 | ❌ 未实现（仅预留接口） |
| **模块四：用户生命周期引擎** | 基于用户行为自动执行个性化触达策略 | ❌ 未实现 |
| **模块五：竞品情报系统** | 采集竞品公开数据，分析后反哺决策与内容 | ⚠️ 部分实现（LLM 模拟分析，无真实采集） |

## 二、BRD 定义的完整交互流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    智营引擎完整交互流程                           │
└─────────────────────────────────────────────────────────────────┘

  [1. 专家初始化]
       ↓
       │ 投放师/运营专家将经验编写为结构化规则表
       │ 注入决策大脑
       ↓
  [2. 大脑决策] ←──────────────────────────────────────┐
       ↓                                                │
       │ 根据目标（拉新/复购）+ 当前数据                  │
       │ 输出：投放策略 + 内容需求单                       │
       ↓                                                │
  [3. 自动执行]                                         │
       ├─→ 内容生成器：生产文/图/视频                     │
       └─→ 投放连接器：RPA 自动建计划、上传素材            │
       ↓                                                │
  [4. 人工审核]                                         │
       ↓                                                │
       │ 所有内容和高风险决策进入审核队列                   │
       │ 专家每日 30-60 分钟快速处理                      │
       │ 驳回附带原因，反馈到提示词库优化                  │
       ↓                                                │
  [5. 消费者触达]                                       │
       ↓                                                │
       │ 广告正常展示，用户下单                           │
       │ 生命周期引擎：自动发送复购提醒、优惠券            │
       ↓                                                │
  [6. 数据闭环] ────────────────────────────────────────┘
       │ 投放效果、用户行为、竞品情报回流大脑
       │ 持续优化模型
       │ 专家干预频率随模型成熟而下降
```

## 三、当前 MVP 实现 vs BRD 要求

### 3.1 模块一：专家决策大脑

**BRD 要求：**
- 输入：投放预算上限、ROI 目标、用户分层规则、产品毛利结构
- 处理：实时接收投放数据、用户行为数据、竞品情报，根据规则 + ML 模型输出决策
- 输出：投放计划参数（出价、人群、预算分配）、内容需求单、用户触达指令

**当前实现：**
```
✅ 已实现：
  - 规则引擎（lib/decision-engine.ts）
    - 预算分配规则（集中 vs 分散）
    - 出价策略（保守/平衡/激进）
    - 人群策略（拉新/复购/品牌）
    - 素材策略（更新频率、形式优先级）
    - 时段策略（高峰时段建议）
  - LLM 增强分析（解读规则输出，给出行动建议）
  - 置信度评估

❌ 未实现：
  - 实时投放数据接入（需要对接千川/聚光 API）
  - 用户行为数据接入
  - 强化学习（以 ROI 为奖励信号）
  - 自动输出投放计划参数（出价、人群包）
```

**交互逻辑：**
```
用户输入：
  - 投放目标（拉新/复购/品牌）
  - 日预算（¥100-10000）
  - 目标 ROI（0.5-10）
  - 产品范围（全部/单品）
       ↓
规则引擎处理：
  - 匹配 5 类规则
  - 输出优先级排序的建议列表
       ↓
LLM 增强：
  - 解读规则结果
  - 给出行动建议 + 时间线
  - 预判风险
       ↓
输出展示：
  - 规则结果卡片（优先级、置信度、预期效果）
  - AI 分析文本
```

### 3.2 模块二：内容生成器

**BRD 要求：**
- 文案生成：调用大模型 API，嵌入专家高转化范例，限定品牌调性和违禁词
- 图片生成：调用图生图 API（Stable Diffusion/可灵）
- 视频生成：商品卡混剪 + AI 脚本 + AI 配音字幕
- 人机协同：内容进入审核队列，专家修改后系统自动优化提示词库

**当前实现：**
```
✅ 已实现：
  - 文案生成（小红书/抖音/微信公众号）
  - 多类型输出（种草文案/视频脚本/图片提示词）
  - 平台风格适配（口语化/emoji/专业等）
  - 审核队列 UI（/review 页面）
  - 导出功能（Markdown/HTML/JSON）
  - 微信公众号草稿上传 API

❌ 未实现：
  - 图片生成 API 对接（Stable Diffusion/可灵）
  - 视频生成（商品卡混剪、AI 配音）
  - 专家修改反馈到提示词库的闭环
  - 违禁词库（当前仅 LLM 自带过滤）
  - 50+ 高转化范例的 Prompt 工程
```

**交互逻辑：**
```
用户输入：
  - 选择产品（烟酰胺精华/头皮精华/玻色因面霜）
  - 选择平台（小红书/抖音/微信）
  - 选择类型（文案/脚本/提示词）
  - 额外要求（可选）
       ↓
AI 生成：
  - 构建系统提示词（平台风格 + 品牌知识）
  - 构建用户提示词（产品信息 + 卖点）
  - 调用 LLM API
       ↓
输出展示：
  - 生成结果预览
  - 导出按钮（文件下载/剪贴板复制）
  - 发布按钮（微信公众号 API）
       ↓
审核队列（/review）：
  - 列表展示所有生成内容
  - 通过/驳回/编辑操作
  - （当前为 UI 展示，未实现反馈闭环）
```

### 3.3 模块三：投放连接器 (RPA)

**BRD 要求：**
- 支持平台：抖音千川、小红书聚光

- 决策联动：接收大脑指令，对 ROI 达标的计划追加预算，低于阈值的自动暂停
- 容灾机制：脚本异常自动告警，24h 内修复

**当前实现：**
```
❌ 完全未实现

预留接口：
  - /settings 页面有平台配置 UI（Cookie/Token 输入框）
  - /api/status 有平台状态检查逻辑
  - lib/types.ts 定义了 PlatformConfig 接口
```

**BRD 要求的交互逻辑（待实现）：**
```
触发条件：
  - 内容审核通过
  - 决策大脑输出投放计划参数
       ↓
RPA 执行：
  - 登录广告平台（Cookie/Token 认证）
  - 创建广告计划（标题、预算、出价、人群）
  - 上传素材（图片/视频/文案）

数据抓取：

  - 写入数据库
  - 回流到决策大脑
       ↓
决策联动：
  - ROI > 阈值 → 追加预算 20%

  - 异常告警 → 通知专家
```

## 四、全链路工作流（Pipeline）交互逻辑

当前 MVP 的核心交互是 **全链路工作流**，串联了 4 个步骤：

```
┌─────────────────────────────────────────────────────────────────┐
│                    全链路工作流交互逻辑                           │
└─────────────────────────────────────────────────────────────────┘

[步骤 1：策略生成] ──→ [步骤 2：竞品分析] ──→ [步骤 3：内容生成] ──→ [步骤 4：审核发布]
     ↓                      ↓                      ↓                      ↓
  输入：                  输入：                  输入：                  输入：
  - 产品信息             - 竞品名称              - 策略（步骤1输出）     - 内容（步骤3输出）
  - 投放目标             - 行业类别              - 竞品分析（步骤2输出） - 策略（步骤1输出）
  - 日预算                                       - 平台                  - 竞品分析（步骤2输出）
  - 目标ROI                                      - 内容类型
       ↓                      ↓                      ↓                      ↓
  输出：                  输出：                  输出：                  输出：
  - 投放策略文本          - 竞品分析文本          - 3组差异化内容         - 质量评分（1-10）
  - 预算分配建议          - 差异化方向            - 平台适配格式          - 违禁词检查
  - 内容方向             - 关键词参考                                     - A/B测试建议
                                                耗时：~60秒              - 投放建议
  耗时：~45秒            耗时：~45秒                                     耗时：~30秒

总耗时：~180秒（3分钟）

交互方式：
  - 单步执行：点击每步按钮，手动触发
  - 一键全流程：点击"🚀 一键运行全流程"，自动串联 4 步
  - 上下文传递：每步输出自动作为下一步输入
  - 结果展示：每步结果独立卡片，支持复制/下载
```

## 六、关键差距与优先级

| 优先级 | 差距 | 影响 | 工作量 |
|--------|------|------|--------|
| **P0** | 投放连接器 (RPA) | 无法自动投放，需手动操作 | 2-3 周 |
| **P0** | 数据回流闭环 | 无法优化模型，专家需手动分析 | 1-2 周 |
| **P1** | 图片/视频生成 | 内容形式单一，竞争力不足 | 1-2 周 |
| **P1** | 竞品数据采集 | 竞品分析基于 LLM 模拟，不够精准 | 2-3 周 |

| **P3** | 违禁词库 | 依赖 LLM 自带过滤，有漏网风险 | 2-3 天 |
| **P3** | 50+ 高转化范例 Prompt | 内容质量依赖 LLM 通用能力 | 1-2 周 |

## 八、总结

### 当前 MVP 的价值

1. **验证了核心交互逻辑**：全链路工作流（策略→竞品→内容→审核）跑通
2. **展示了 AI 能力**：真实 LLM 调用，3 分钟生成完整营销内容
3. **证明了可行性**：决策引擎 + LLM 增强的架构可行
4. **降低了演示门槛**：浏览器直接使用，无需安装

### 距离 BRD 完整产品的差距

1. **执行层缺失**：RPA 投放连接器未实现，无法自动投放
2. **数据层缺失**：无真实数据回流，无法自动优化
3. **触达层缺失**：用户生命周期引擎未实现，无法自动复购
4. **情报层弱化**：竞品分析基于 LLM 模拟，无真实采集

### 建议的下一步

1. **优先实现 RPA 投放连接器**（千川/聚光），打通"生成→投放"闭环
2. **接入投放数据 API**，实现数据回流，让决策大脑能基于真实数据优化
3. **完善审核反馈闭环**，专家修改内容后自动优化提示词库
